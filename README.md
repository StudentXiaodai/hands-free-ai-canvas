# 产品需求与使用文档 (PRD)

## 产品概述
**项目名称**：小豆语音绘图助理 (Hands-Free Speech AI Canvas)
**核心亮点**：完全免手操作的纯语音控制绘图应用。用户仅通过语音与系统对话，即可实现从零开始的创意绘画、画面迭代和文件导出。

## 本地启动与部署指南

### 1. 环境准备
确保本机已安装 **Node.js** (推荐 v18+ 版本) 和 **npm**。

### 2. 配置环境变量
项目根目录下包含 `.env.example` 文件。在运行前，请先创建配置文件：
1. 复制 `.env.example` 并重命名为 `.env`。
2. 打开 `.env` 文件，填入您已申请的火山引擎方舟平台信息：
```env
VOLC_ARK_API_KEY="your_api_key_here"
VOLC_ENDPOINT_TEXT="ep-xxxx"
VOLC_ENDPOINT_IMAGE="ep-xxxx"
```

### 3. 安装依赖与启动项目
在项目根目录终端依次执行以下命令：

```bash
# 1. 安装核心依赖
npm install

# 2. 启动开发环境服务器
npm run dev
```
提示：启动成功后，在浏览器中访问控制台打印的地址（通常为 `http://localhost:3000`）即可体验系统。

### 4. 生产环境构建
```bash
# 编译全栈项目
npm run build

# 启动生产版本服务
npm run start
```

## 快速上手指南

### 1. 唤醒与交互
- **启动**：点击页面中央的“开启体验”按钮进入操作界面，点击底部面板的麦克风图标启动录音（开启后界面右上角会提示“麦克风已开启”）。
- **语音输入**：直接说出你想画的内容。例如：“画一只在太空遨游的小狗”。
- **智能等待UI**：在处理和生图过程中（指示器显示为橙色并旋转），请耐心等待。系统后台正在调用大模型将您的语音指令与历史语境融合，并渲染最新画面。

### 2. 系统核心控制指令 (快捷词拦截)
为防止大模型对于系统操作发生幻觉，系统内置了最高优先级的**固定快捷指令**，当你的语音命中了特定的前缀或相关词汇时，直接执行系统事件：
- **🖌️ 清空画布**：说出 **`“系统清空画布”`**、**`“重新开始”`** 或 **`“全部删掉”`**，系统会清空当前画作并抹除上下文记忆。
- **💾 保存图片**：说出 **`“系统保存作品”`**、**`“系统导出图片”`** 或 **`“把图存下来”`**，系统将自动触发浏览器下载，将成品妥善保存到本地。
- **📐 切换比例**：说出 **`“系统切换比例 16比9”`**、**`“设置成一比一”`** 等，可动态调整后续生图的宽高比。

### 3. 多轮迭代与主动追问 (AI Agent 对话状态机)
- **渐进式迭代**：若您对当前图片希望继续打磨，无需重新说一大串背景，直接说出增量指令：“加一只蝴蝶在向日葵上”、“把风格换成油画”。后端的决策中枢会自动合并前文，生成一张融合了所有要求的全新 Prompt 进行整图重建。
- **主动触发澄清**：如果您的指令缺乏核心名词或难以提炼重点（如：“再改一下”、“加点东西”、“重新画”），AI判定意图模糊，将会暂停生图并主动使用语音 (TTS) 向您追问：“请问具体想画什么？”，此时您可以补充关键信息。
- **自动丢弃噪音**：误碰麦克风或捕捉到的无明确语义的呼吸声、单字，会被决策中枢判定为 `NOISE`，系统静默丢弃并退回防打扰的 `IDLE` 状态。

## 系统配置备忘
在使用 API 之前，请确认后台或本地已配置好大语言模型与文生图模型的火山引擎接入环境变量：
- `VOLC_ARK_API_KEY`: 火山引擎火山方舟平台 API Key。
- `VOLC_ENDPOINT_TEXT`: 用于分析推理的文本大模型 Endpoint (如 `ep-xxx`)。
- `VOLC_ENDPOINT_IMAGE`: 调用的豆包文生图模型 Endpoint (如 `ep-xxx`)。
